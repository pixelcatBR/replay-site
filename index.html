Introduction

Replay is a command-line tool for creating portable project backups.

With a single command, you save the entire folder structure and file contents into an .rpl file. You can then restore that snapshot on any machine, in any directory, using a simple command.

Ideal for developers who need to save project snapshots, share code without relying on Git, or quickly and efficiently migrate projects between machines.

Lightweight, fast, and dependency-free.

Why Replay?

Unlike traditional backup tools like tar or zip, Replay generates a human-readable .rpl file that preserves the complete project structure, including folder hierarchy and file contents.

It's perfect for creating restore points before major refactoring, archiving specific versions, or sharing projects with teammates who don't have access to your repository.

No complex configurations, no external dependencies, and no vendor lock-in.

Features

Complete Backup - Captures the entire project, including all subdirectories and files.

Instant Restore - Recreate your project anywhere with a single command.

Portable - A single .rpl file contains everything. Take it anywhere.

Simple & Fast - No setup required. Just Python 3 and one command.

Human-Readable - The .rpl format is plain text, easy to inspect or modify.

How It Works

Replay scans your project recursively, collecting all directories and files. It saves the relative paths and file contents into a structured .rpl file.

When restoring, it reads the .rpl file and recreates the entire structure in your target directory, creating folders and files exactly as they were.

Everything is preserved - your code, configuration files, documentation, and any other assets.

Quick Example

Creating a backup:

replay -d ./myproject -n backup -s ./myproject

Restoring a backup:

replay -d ./restored -n backup -c

That's it. No configuration files, no environment variables, no setup.

Use Cases

Snapshots - Save a version of your project before making major changes.

Sharing - Send a complete project to a colleague without using Git or cloud storage.

Migration - Move projects between different machines or environments.

Archiving - Keep historical versions of your projects in a compact format.

CI/CD - Use in pipelines to create backup points during automated workflows.

Installation

curl -sSL https://raw.githubusercontent.com/pixelcatBR/replay/main/setup.sh | bash

Requires Python 3.6+ and works only on Linux.
